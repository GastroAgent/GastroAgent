# HALT幻觉检测方法实现说明

## 概述

本代码实现了基于HALT（Hallucination Detection）方法的轻量级幻觉检测系统，核心思想是利用模型中间层的隐藏状态来预测幻觉风险。

## 核心原理

### 1. 中间层隐藏状态提取
- **假设**: LLM在生成答案之前的内部表示中已经编码了关于自身不确定性的信号
- **关键发现**: 中间层（而非最后一层）的表示最能有效检测幻觉
- **实现**: 提取模型中间层（默认50%位置）的隐藏状态作为输入特征

### 2. 轻量级探针模型
- **架构**: 简单的MLP分类器（3层全连接网络）
- **输入**: 仅使用问题（Query）的中间层隐藏状态，不依赖生成的答案
- **输出**: 幻觉风险分数 [0, 1]
- **优势**:
  - 计算量极小（<1% 生成成本）
  - 可与推理过程完全并行运行（零延迟）

### 3. 代理路由机制（Agentic Routing）
- **低风险**: 探针预测低幻觉风险 → 直接输出当前模型结果（零延迟）
- **高风险**: 探针预测高幻觉风险 → 路由到验证管道
  - 选项1: 使用更强大的模型
  - 选项2: 检索增强生成（RAG）
  - 选项3: 交叉验证

## 文件说明

### 1. step1_model_inference.py（主推理脚本）
修改后的推理脚本，集成了HALT检测功能。

**主要修改**:
- 添加了`HALTProbeModel`类定义
- 添加了`extract_middle_layer_hidden_states()`函数
- 添加了`halt_detect_hallucination()`函数
- 在推理流程中集成HALT检测和代理路由

**新增配置参数**:
```python
HALT_ENABLED = True                    # 是否启用HALT检测
HALT_MIDDLE_LAYER_RATIO = 0.5          # 使用中间层位置
HALT_PROBE_HIDDEN_DIM = 256            # 探针模型隐藏层维度
HALT_RISK_THRESHOLD = 0.5              # 幻觉风险阈值
HALT_PROBE_MODEL_PATH = None           # 预训练探针模型路径
ENABLE_AGENTIC_ROUTING = True          # 是否启用代理路由
```

**新增输出字段**:
- `halt_risk_score`: HALT预测的幻觉风险分数 [0, 1]
- `halt_high_risk`: 是否为高风险
- `halt_needs_verification`: 是否需要路由到验证管道
- `halt_layer_ratio`: 使用的中间层位置

### 2. train_halt_probe.py（探针训练脚本）
用于训练HALT探针模型的独立脚本。

**训练数据要求**:
每个样本需要包含：
- `middle_layer_hidden`: 中间层隐藏状态（向量）
- `is_correct`: 答案是否正确（布尔值）

**训练流程**:
1. 加载带标注的训练/验证数据
2. 初始化探针模型
3. 使用BCE损失训练二分类器
4. 保存最佳F1分数的模型

## 关键优势

1. **零延迟检测**: 探针计算与生成并行，不增加延迟
2. **轻量级**: 探针模型参数量极小，计算成本<1%
3. **无需答案**: 仅基于问题的中间层表示，不依赖生成结果
4. **中间层优势**: 实验证明中间层比最后一层更有效
5. **灵活路由**: 可根据风险分数动态选择验证策略

## 实验建议

### 1. 中间层位置调优
尝试不同的`HALT_MIDDLE_LAYER_RATIO`值：
- 0.3: 较浅层（更早期的语义理解）
- 0.5: 中间层（推荐）
- 0.7: 较深层（更接近输出）

### 2. 风险阈值调优
根据业务需求调整`HALT_RISK_THRESHOLD`：
- 较低阈值（0.3）: 更保守，更多样本路由验证
- 较高阈值（0.7）: 更激进，只验证高风险样本

### 3. 探针架构调优
可以尝试：
- 更深的MLP（4-5层）
- 添加注意力机制
- 使用Transformer探针

## 注意事项

1. **探针训练数据质量**: 需要足够的正负样本（正确/错误答案）
2. **领域适应性**: 探针可能需要针对特定领域重新训练
3. **模型兼容性**: 确保模型支持`output_hidden_states=True`
4. **内存开销**: 提取隐藏状态会增加一定内存消耗

## 参考文献

HALT方法基于以下核心思想：
- 模型内部表示编码了不确定性信号
- 中间层比最后一层更有效
- 轻量级探针可实现零延迟检测
- 代理路由可平衡效率和准确性

## 后续改进方向

1. **多层融合**: 结合多个中间层的信息
2. **动态层选择**: 根据问题类型自适应选择最优层
3. **在线学习**: 根据验证结果持续更新探针
4. **集成验证**: 结合多种验证策略（强模型+RAG+交叉验证）
